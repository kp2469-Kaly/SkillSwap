# SkillSwap - Validation

# 1. Description

SkillSwap is a peer-to-peer skill exchange platform that empowers users to trade knowledge within their local or virtual communities. Instead of paying money, users can teach a skill they excel at in exchange for learning a new skill from someone else. The system focuses on human-centered learning and community interaction, offering both online and in-person sessions.

The value proposition lies in its affordability, flexibility, and personalized learning experience. Our Minimum Viable Product (MVP) targets individuals who are eager to learn or share skills such as guitar, coding, painting, or cooking. Core features include user profile creation, skill matching, session scheduling, expert consultation, and notification systems.

# 2. User Evaluation Script

## User Tasks

- Task 1: Register a new account, set up profile with skills you can teach and learn.
- Task 2: Browse matches based on selected skills and schedule a session with a matched user.
- Task 3: Respond to a session request from another user (accept/decline).
- Task 4: Join a group and post a message related to the skill topic.

## Data Collection Methods

- Time taken by users to complete each task with minimal instructions
- Number of support prompts or clarifying questions asked by users during the session
- Behavioral observations and Spoken feedback
- Post-task questionnaire responses reflecting user impressions and system alignment

## Post-Task Interview Questions

- What was your initial impression of the interface when you logged in?
- Were the options for choosing teaching and learning skills clear and easy to use?
- Did the matching suggestions feel relevant to your chosen preferences?
- How did the scheduling process feel in terms of convenience?
- What features made you feel most useful?
- Was anything about the expert booking process confusing?
- What parts of the dashboard or navigation felt unnecessary or hard to find?
- Do you actually feel this platform could help you learn or teach a skill regularly?
- How would you describe your overall experience?

## Script Structure

- Welcome & Introduction (5 min)
- Task 1: Profile setup (5 min)
- Task 2: Skill Matching and session scheduling (10 min)
- Task 3: Session handling (5 min)
- Task 4: Group participation (3-5 min)
- Task 5: Expert booking (2 min)
- Post-task Interview (15 min)
- Wrap-up (5 min)

# 3. Results Summary

## Participant Demographics

We conducted evaluations with four participants of different segments of our target audience. Each participant interacted with the SkillSwap system and completed the key user tasks defined in our evaluation script.

- Sarah, 26 -- Freelance graphic designer based in London. Interested in bartering her design skills for learning guitar. Represents the socially-driven learner ideal for SkillSwap.
- Akhil, 23 -- Computer Science undergraduate from India. More focused on structured and credentialed learning paths. His skepticism provides insight into limitations of the barter model for technical learners.
- Naveen, 21 -- Software engineer preparing for AWS certification. Technically proficient and open to peer learning, especially for soft skills like communication and presentation.
- Sindhu, 40 -- Music tutor from Leeds. Active in local learning circles, already familiar with peer teaching. Embodies the in-person community exchange user segment.

## Qualitative Findings

### Common Feedback Themes:

- The interface was generally well-received, especially the dashboard layout.
- Most users liked the concept of exchanging skills locally.
- Skill matching was intuitive but needed better filtering options.
- The group feature caused some confusion among users unfamiliar with public discussion interfaces.
- One participant (Akhil) did not feel comfortable bartering as a learning method, preferring traditional education formats.
- "I like that I can teach something and learn something in return, it's more personal." -- Sarah
- "Honestly, I don't think I'd use this. I prefer formal learning platforms like Coursera or Udemy." -- Akhil
- Akhil skipped the profile setup halfway, saying he didn't see long-term value.
- Sindhu tried to filter expert sessions by price but found the filters lacking.
- All the users were able to create their profile with ease.

# 4. Analysis and Reflection

The evaluation sessions provided meaningful insight into how users interact with the SkillSwap platform and how well it delivers on its value proposition.

## Value Proposition

For the majority of participants, SkillSwap successfully delivered on its core promise by facilitating skill-based barter interactions. Three out of four participants noted the community-based human engagement model and learning dynamic as a valuable addition to traditional pedagogies. On the other hand, one participant leaned towards more conventional, structured, certified formats and showed almost no interest in informal, peer-driven exchange-based learning. It implies that the platform is unlikely to engage all users across the spectrum of pedagogical and educational approaches, at least in more technical or academic disciplines.

## What Worked Well

- Profile creation and dashboard navigation were intuitive for all users.
- Skill matching received strong usability scores due to clear categorization and suggestions.
- Expert booking was generally perceived as seamless.
- Visual layout and design were praised, especially the clarity of session scheduling.

## Areas for Improvement

- Group feature created mild confusion and needs improved onboarding or tooltips.
- Filtering options for expert sessions should be more comprehensive.
- Some users (e.g., Akhil) were hesitant about the barter model; the platform could benefit from hybrid features like optional paid expert sessions or certifications.
- Clarification of public vs. private interactions in groups is needed to ensure privacy expectations.

# 5. Proposed Improvements

Based on feedback from the user evaluation sessions, we propose the following changes for the next iteration of SkillSwap:

## Top 3 Priority Changes

1. Redesign the group feature with guided walkthroughs and better categorization.
2. Enhance expert filtering options (by price, availability, skill rating).
3. Introduce hybrid learning formats -- allow users to optionally access certified courses or structured content.

## Assumptions to Revisit

- The assumption that all users will embrace peer-to-peer learning may not hold for users seeking formal education or credentials.
- Community-led learning is beneficial for some demographics but needs additional structure or incentives for broader appeal.

## Plans for Additional Validation Activities

- A/B testing interface elements like the expert search filters.
- Conduct interviews with educators and certified tutors to gauge professional integration opportunities.
- Expand evaluation to non-technical users and non-urban areas to ensure accessibility and inclusivity.
